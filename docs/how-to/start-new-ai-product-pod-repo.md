# How to Start a New AI Product Pod Repository

## Purpose

Bootstrap a new Pod repository with AI-first engineering and documentation
standards from day one.

## Setup Steps

1. Create repository skeleton:
   - `src/`, `tests/`, `prompts/`, `evals/`, `docs/`, `.cursor/rules/`
2. Add baseline docs:
   - initiative brief
   - operating cadence
   - release readiness checklist
3. Add Cursor project rules for:
   - grounding to local docs
   - markdown standards
   - team context
4. Add evaluation baseline:
   - initial quality checks
   - target thresholds
5. Add runbooks:
   - deploy
   - rollback
   - incident triage

## Exit Criteria

- Team can ship one vertical slice with docs + evals + monitoring.
- Chat answers are citation-grounded in local markdown.
