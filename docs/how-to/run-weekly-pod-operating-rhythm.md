# How to Run the Weekly Pod Operating Rhythm

## Purpose

Create predictable execution for a fast-moving AI Pod.

## Weekly Sequence

1. Monday planning (45-60 min)
   - Confirm this week's experiment goals.
   - Define success/fail metrics.
2. Midweek checkpoint (30 min)
   - Review blockers and decision needs.
   - Re-scope quickly if evidence changes.
3. Friday review (45 min)
   - Demo current state.
   - Capture metrics and lessons.
   - Document decisions and next bets.

## Required Artifacts Per Week

- 1 updated scope/status note in `docs/reference/processes/`.
- 1 technical update in `docs/reference/engineering/`.
- 1 runbook or how-to update in `docs/how-to/`.

## Escalation Triggers

- A critical dependency blocked for >3 business days.
- Evaluation failure on a core quality metric.
- Security/compliance uncertainty for production-bound features.
