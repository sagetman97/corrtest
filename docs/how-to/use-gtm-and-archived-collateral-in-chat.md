# How to Use GTM and Archived Collateral in Cursor Chat

## Purpose

Use new GTM materials effectively while avoiding stale or outdated claims from
archived collateral.

## Source Priority

1. First: `docs/reference/**` and other canonical docs.
2. Second: `docs/GTM -20260413T175856Z-3-001/**` for current planning context.
3. Last: `docs/(DNU) ARCHIVED sales collateral/**` only for historical context.

## Prompt Pattern

Use prompts like:

- "Answer using canonical docs first, then GTM files if needed."
- "Only use archived collateral for historical comparison."
- "Flag if any claim is based on archived or potentially outdated materials."

## Verification Steps

1. Ask for cited file paths.
2. Ask whether source is canonical/current/historical.
3. For high-impact claims, request cross-check against canonical reference docs.

## Anti-Patterns

- Treating archived decks as current product policy.
- Mixing current and historical claims without labeling time context.
- Using pricing terms from collateral without explicit validation.
