# How to Create an Initiative Brief

## Purpose

Standardize initiative framing so the Pod can decide quickly and execute with
clarity.

## Steps

1. Copy `docs/templates/initiative-brief-template.md`.
2. Name file with lowercase hyphen format.
3. Fill problem, outcomes, scope, and feasibility sections.
4. Add explicit success and stop criteria.
5. Link relevant dependencies and risks.
6. Review with Pod and finalize DRI ownership.

## Quality Checks

- Scope fits 3-6 month horizon.
- Metrics are measurable and time-bound.
- Out-of-scope list is explicit.
- Risks include security/compliance and dependency impact.
