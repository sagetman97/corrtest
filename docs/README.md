# Docs Folder Guide

Use this folder for all persistent Polly documentation.

## Folder Map

- `tutorials/`: Learn a workflow from start to finish.
- `how-to/`: Practical task playbooks and runbooks.
- `reference/`: Stable facts, inventories, and definitions.
- `explanation/`: Deep context and rationale.
- `templates/`: Reusable markdown templates.
- `sources/`: Raw source artifacts and source notes.
- `glossary.md`: Shared vocabulary.
- `GTM -20260413T175856Z-3-001/`: Current planning input materials (Tier 2).
- `(DNU) ARCHIVED sales collateral/`: Historical/non-default context (Tier 3).

## Authoring Rule of Thumb

Before creating a new file, ask:

- Is this teaching someone? Put it in `tutorials/`.
- Is this solving a task? Put it in `how-to/`.
- Is this factual truth? Put it in `reference/`.
- Is this conceptual background? Put it in `explanation/`.

## Alignment Standard

- Use templates in `docs/templates/`.
- Follow `docs/reference/documentation-quality-checklist.md`.
- Maintain the process docs in `docs/reference/processes/`.
- Follow source-tiering in `docs/reference/company/source-tiering-and-reliability.md`.
