# First 30 Days as AI Solutions Manager

## Purpose

Provide a guided onboarding sequence to stand up and run the new AI Pod.

## Week 1: Foundations

1. Review core docs:
   - `docs/reference/knowledge-operations-framework.md`
   - `docs/explanation/pod-operating-model.md`
   - `docs/reference/team/ai-solutions-manager-charter.md`
2. Confirm tool landscape using:
   - `docs/reference/tools/tooling-integration-matrix.md`
3. Establish team rhythm:
   - `docs/reference/team/operating-cadence.md`

## Week 2: Execution Baseline

1. Start first initiative brief using template.
2. Stand up backlog and scope in Jira.
3. Define first evaluation metrics and release checklist.

## Week 3: Operating Loop

1. Run weekly rhythm and decision logging.
2. Ingest key tool signals into markdown.
3. Validate chat response quality with citation checks.

## Week 4: Hardening

1. Review metrics and blocker trends.
2. Refine standards/templates from real execution pain points.
3. Document next 60-day priorities.

## Completion Criteria

- One active initiative with clear DRI and metrics.
- One working ingestion-to-doc loop.
- One completed release-readiness review (even for internal pilot).
- Chat answers consistently grounded in repository docs.
