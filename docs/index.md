## Information Architecture

This repository uses a Diataxis-inspired layout:

- Tutorials: Learning by doing.
- How-to guides: Solve a specific task.
- Reference: Facts and technical truth.
- Explanation: Conceptual understanding.

## Writing and Quality

- Use templates from `docs/templates/`.
- Keep docs concise and cross-linked.
- Prefer updating existing pages over creating duplicates.
- Apply metadata (owner, source, last reviewed, confidence) on critical docs.
- Repository conventions: `CONTRIBUTING.md` (root); Python tooling: `requirements.txt`.
