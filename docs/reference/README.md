# Reference

Use reference docs for objective, stable facts.

## Characteristics

- Accurate and concise.
- Minimal interpretation.
- Optimized for lookup.

## Suggested Subfolders

- `people/`: Teams, reporting lines, ownership.
- `tools/`: Systems and access model.
- `processes/`: Canonical checklists and official steps.
- `team/`: Role charters, scorecards, and operating cadence.
- `company/`: Company-level context and strategic framing.
- `engineering/`: Standards, quality, and delivery practices.
