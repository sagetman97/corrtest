# Documentation Lifecycle

## Metadata

- Owner: AI Solutions Manager
- Last reviewed: 2026-04-13
- Confidence: High

## Purpose

Standardize how docs are created, validated, updated, and retired.

## Lifecycle Stages

1. Draft
   - Initial content captured.
   - Sources and owner added.
2. Active
   - Used by team workflows and chat answers.
   - Reviewed on a recurring cadence.
3. Watch
   - Content still useful but likely to change soon.
   - Add explicit review date.
4. Archived
   - Replaced or obsolete.
   - Keep historical context but remove from primary index paths.

## Required Metadata for Active Docs

- Owner
- Last reviewed date
- Source(s)
- Confidence level (where uncertainty exists)

## Review Cadence

- Critical process docs: every 30 days.
- Tool profiles and team operating docs: every 60 days.
- Conceptual explanation docs: every 90 days.

## Change Triggers

- Org or ownership change
- Process change
- Security/compliance requirement updates
- Repeated chat answers with low confidence or conflicting references
