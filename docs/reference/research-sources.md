# Research Sources and Takeaways

This page captures external guidance used to design this repository.

## Information Architecture

- Diataxis Start Here:
  <https://diataxis.fr/start-here/>
  - Use four doc types: tutorials, how-to guides, reference, explanation.
  - Keep intent boundaries clean to reduce confusion.

## Markdown Quality

- Google Markdown Style Guide:
  <https://google.github.io/styleguide/docguide/style.html>
  - Keep markdown simple, readable, and maintainable.
  - Use clear heading structure and consistent formatting.
- Markdown Guide Basic Syntax:
  <https://www.markdownguide.org/basic-syntax/>
  - Use portable syntax patterns for broad compatibility.

## Repository Practices

- GitHub Repository Best Practices:
  <https://docs.github.com/en/repositories/creating-and-managing-repositories/best-practices-for-repositories>
  - Include a clear README.
  - Use healthy repository hygiene and secure workflows.

## Cursor Rules and Agent Configuration

- Cursor Rules Documentation:
  <https://cursor.com/docs/rules>
  - Use project rules in `.cursor/rules`.
  - Use metadata (`description`, `alwaysApply`, `globs`) for scoped behavior.
  - Use `AGENTS.md` for straightforward global instructions.

## Polly Public Context
