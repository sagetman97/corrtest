# Pod Metrics and Scorecard

## Metadata

- Owner: AI Solutions Manager
- Last reviewed: 2026-04-13
- Confidence: High (operating baseline)

## Purpose

Track outcome and execution quality for the AI Pod across rapid 3-6 month
product cycles.

## North-Star Metrics

- Time-to-first-prototype (days)
- Time-to-pilot-ready (days)
- Pilot outcome conversion rate

## Delivery Metrics

- Weekly scope completion rate
- PR lead time (open to merge)
- Work-item cycle time (Jira start to done)
- Blocker age (median days)

## Quality and Reliability Metrics

- Defect escape rate
- Production incident frequency
- Mean time to detect and recover
- AI evaluation pass rate on core tasks

## Learning Metrics

- Number of validated assumptions per month
- Number of invalidated assumptions per month
- Reuse of shared components/prompts/evals

## Reporting Rhythm

- Weekly operating review
- Monthly milestone review
- Quarterly retrospective and model reset

## Data Sources

- Jira
- GitHub
- Datadog/Sentry/CircleCI
- Internal scorecards and stakeholder feedback
