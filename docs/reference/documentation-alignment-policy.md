# Documentation Alignment Policy

## Purpose

Prevent drift and contradictions across the knowledge base as it grows.

## Alignment Rules

- Update canonical pages before creating new overlapping docs.
- Link new pages from relevant section README files.
- When replacing a page, mark old page status and point to replacement.
- Keep terminology aligned with `docs/glossary.md`.
- Do not treat `(DNU) ARCHIVED sales collateral` as default source for current
  claims.

## Conflict Resolution

If two docs conflict:

1. Mark conflict explicitly in both pages.
2. Use decision log to record the resolution.
3. Update stale page with corrected guidance.

## Audit Cadence

- Monthly alignment spot-check across top 20 most-used pages.
- Quarterly full pass on process and standards pages.

## Ownership

- Each canonical page must have an owner.
- Section owners maintain README/index links for discoverability.
