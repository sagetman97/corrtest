# Knowledge Operations Framework

## Purpose

Define how this repository stays accurate, current, and highly usable for
Cursor Chat and human collaborators.

## Core Principles

- Local markdown is the long-term source of truth for assistant answers.
- Every high-impact claim should be traceable to a source.
- Documentation quality is part of delivery quality.
- Fast updates are better than perfect but stale documentation.

## Content Model

- `tutorials/`: Learning sequences.
- `how-to/`: Task execution guides.
- `reference/`: Facts, inventories, standards, and process truth.
- `explanation/`: Mental models and rationale.

## Knowledge Flow

1. Source signal appears (tool update, meeting, decision, incident).
2. Triage importance and confidence.
3. Normalize to markdown in the right section.
4. Link related pages and glossary terms.
5. Use in chat answers with citations.
6. Review and refresh on schedule.

## Quality Gates

- Owner assigned
- Last reviewed date present
- Source(s) listed
- Confidence level included for uncertain domains
- Cross-links added to adjacent pages

## Source Priority for This Repository

- Canonical operational truth: `docs/reference/**`, `docs/how-to/**`,
  `docs/explanation/**`
- Current planning inputs: `docs/GTM -20260413T175856Z-3-001/**`
- Historical context only: `docs/(DNU) ARCHIVED sales collateral/**`

## Operating Cadence

- Weekly: update active initiative/process pages.
- Monthly: review tool profiles and runbooks.
- Quarterly: archive stale pages and refactor structure as needed.
